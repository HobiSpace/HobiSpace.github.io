UICollectionViewLayout使用详解

发表于 2016-03-12   |   分类于 iOS   |  

UICollectionViewLayout

优点

自从会写了UICollectionViewLayout之后，我就再也没有用过TableView，原因很简单，CollectionView由于Layout的存在，可定制性更加强，万一以后一旦有需求变更或者增添需要加些酷炫的效果，CollectionView无疑更加优秀。但是如果只是很简单的列表的话，也可以直接使用TableView，开发效率会更加高。

下面就着重讲一下CollectionView的layout：

简介

Inherits From:
NSObject
－UICollectionViewLayout
－－UICollectionViewFlowLayout

UICollectionViewLayout是一个抽象类，负责定义抽象的接口，CollectionViewFlowLayout是具体实现的方法，系统提供的类用于实现瀑布流Layout布局。FlowLayout我就不多说了，UIKit已经封装好了Delegate让我们去设置相关的属性。下面主要讲讲如何自定义一个Layout。

具体实现

我们在自定义CollectionViewLayout时，需要新建一个UICollectionViewLayout的子类。
需要实现方法和相关参数属性:

UICollectionViewLayoutAttributes

先说布局属性，看头文件

有没有感觉跟我们的UIView中的属性类似，其实这个UICollectionViewLayoutAttributes就是针对我们的CollectionViewCell中的这些UIView的属性的一个存储，在生成我们的Cell时，系统会自动把UICollectionViewLayoutAttributes中的这些属性和CollectionViewCell中的UIView属性进行绑定。

- (void)prepareLayout;

官方注释:
// The collection view calls -prepareLayout once at its first layout as the first message to the layout instance.
// The collection view calls -prepareLayout again after layout is invalidated and before requerying the layout information.
// Subclasses should always call super if they override.
该方法会在每一次生成新的Layout时调用一次，包括第一次生成Layout和Invalidate后都会调用。子类重写该方法必须调用 [super prepareLayout]。我自己喜欢在prepareLayout做一些数据绑定，参数设置，甚至，如果我的layout的Attribute数据是在每一次生成后都固定不变的，我会在prepareLayout就直接调用- (nullable UICollectionViewLayoutAttributes )layoutAttributesForItemAtIndexPath:(NSIndexPath )indexPath;去生成我的布局数据，因为这样可以减少我在滑动过程中的计算。可以参考我下面demo代码或者我的微博项目

- (nullable NSArray<__kindof UICollectionViewLayoutAttributes > )layoutAttributesForElementsInRect:(CGRect)rect;

返回一个数组，数据成员为UICollectionViewLayoutAttributes类或者子类，传入参数Rect是可视Rect，具体上图大家就会明白Rect参数

我在别的文章有看到别人做法是，写个辅助方法，获取Rect中的可见IndexArray，利用这个IndexArray生成对应的UICollectionViewLayoutAttributes，但是这种方法我觉得在LayoutAttribute属性不固定的情况下很适合使用，但是像微博这样的，一旦微博生成了，长度高度都是固定的话，我认为不适合每次都这样计算，把计算放在preparelayout里面，该方法只返回一个全部数据的数组就可以了，没必要每次
layoutAttributesForElementsInRect都去计算（个人见解，有待论证）

- (nullable UICollectionViewLayoutAttributes )layoutAttributesForItemAtIndexPath:(NSIndexPath )indexPath;

这个方法根据IndexPath去生成一个UICollectionViewLayoutAttributes，更多的是主动调用，可以在prepareLayout中调用，也可以在layoutAttributesForElementsInRect，看自己需求吧。具体Atrribute属性的赋值都在这个方法内完成。

- (BOOL)shouldInvalidateLayoutForBoundsChange:(CGRect)newBounds;

根据collectionView的bound判断是否需要Invalidate，一般是用于屏幕旋转时的layout重新适配。千万不要直接返回yes，因为这样你的collectionView会不断生成新的layout，重复了大量的计算，影响性能。

- (CGSize)collectionViewContentSize;

这个不用说了，就是返回collectionview的ContentSize。
